# LoqaAudioBridge Example App

This example demonstrates integration of `@loqalabs/loqa-audio-bridge` in an Expo managed project.

## Structure

```
example/
├── app.json           # Expo configuration with microphone permissions
├── package.json       # Dependencies (includes file:.. reference to parent module)
├── App.tsx            # Main app component (to be implemented in Story 3.4)
├── ios/               # iOS native project (generated by prebuild)
├── android/           # Android native project (generated by prebuild)
└── assets/            # App assets
```

## Development Commands

- `npm install` - Install dependencies
- `npx expo prebuild` - Generate native projects
- `npx expo run:ios` - Build and run on iOS simulator
- `npx expo run:android` - Build and run on Android emulator
- `npx expo start` - Start Metro bundler

## Module Dependency

This example uses a local file dependency to the parent module:

```json
{
  "dependencies": {
    "@loqalabs/loqa-audio-bridge": "file:.."
  }
}
```

This allows testing changes to the module without publishing to npm.

## Permissions

### iOS
Microphone access is configured via `NSMicrophoneUsageDescription` in app.json:
```json
{
  "ios": {
    "infoPlist": {
      "NSMicrophoneUsageDescription": "This app needs microphone access to demonstrate audio streaming."
    }
  }
}
```

### Android
Microphone permission is declared in app.json:
```json
{
  "android": {
    "permissions": ["RECORD_AUDIO"]
  }
}
```
